Final Project for Human-Computer Interaction Course

Conducted needfinding and heuristic evaluation of Ed Discussion, the online course forum used in Georgia Tech’s OMSCS program. Identified gaps in feedback for students awaiting TA responses and proposed interface enhancements including read receipts and response timers. Designed wireframes and conducted qualitative user testing. Findings revealed strong user support for read receipts but confusion around timer icons, leading to clear next steps for refinement. The project addressed principles of discoverability, feedback, and cognitive load reduction.

Supervised Learning Assignment

Compared K-Nearest Neighbors (KNN), Artificial Neural Networks (ANN), and Support Vector Machines (SVM) on two distinct datasets: a small binary classification dataset (Seeds) and a large, multi-class dataset (Diamonds). Explored how dataset complexity impacts model performance, scalability, and generalization. Tuned hyperparameters (e.g., neighbors, layers, learning rate, kernel type), plotted learning/validation curves, and analyzed F1 score trends. Found that simpler models like KNN excelled on clean, low-dimensional data, while complex datasets like Diamonds required more careful preprocessing and tuning. Unexpectedly, neural networks performed best on the simpler dataset, highlighting tradeoffs between data complexity and model design.

UX Research and Wireframe Prototype for a Synchronous Student Chat Tool

Online students often struggle with isolation and limited peer interaction. Through surveys, interviews, and a literature review, I explored how synchronous communication could improve student experiences in Georgia Tech’s OMSCS program. Using insights from 40+ survey participants, I designed a wireframe prototype for a lightweight, easily accessible live chat tool integrated into course platforms like Canvas or Piazza. The project showcases my ability to conduct UX research, analyze user needs, and propose thoughtful, community-centered design solutions.
